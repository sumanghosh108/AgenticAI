# Security Policy

## Supported Versions

AgenticAI is an actively developed application. We currently provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1   | :x:                |

## Reporting a Vulnerability

Security is a high priority for the AgenticAI project. We appreciate your help in disclosing any vulnerabilities you might find responsibly.

If you discover a security vulnerability within AgenticAI (e.g., exposed API keys, prompt injection vulnerabilities, insecure database configurations, unauthorized access in Streamlit, etc.), please follow these steps:

1.  **Raise a GitHub Issue.** Create a new issue describing the vulnerability, its potential impact, and clear steps to reproduce it.
2.  **Submit a Pull Request (PR) with the Fix.** If you have solved the bug, please create a Pull Request against the `main` branch. Your PR must include:
    *   **Documentation:** Clear instructions on the fix and how to test it.
    *   **Screenshots:** Visual evidence showing the issue occurring *before* your fix, and the behavior *after* your fix is applied.
3.  **Review & Deployment.** A project maintainer will review your Pull Request. Once approved and merged, the fix will be deployed in the next appropriate release.

### What to Expect

*   **Acknowledgment:** You can expect an initial response acknowledging your report within 48 hours.
*   **Assessment:** We will assess the vulnerability and determine the severity and scope of the issue. We may reach out for further clarification.
*   **Resolution:** If the vulnerability is confirmed, we will work on a patch. We aim to release security fixes as quickly as possible.
*   **Disclosure:** Once the vulnerability is resolved and an update is released, we will publicly disclose the issue and credit you for the discovery (unless you prefer to remain anonymous).

Thank you for helping keep AgenticAI secure!
